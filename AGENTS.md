# Файлы и документы — единое правило

**Любой файл на сайте выводится через `DocCard` / `DocCards`
(`src/components/sveden/`). Своей вёрстки карточки файла быть не должно.**
Это уже случалось трижды: каждый раз получался «почти такой же» вид, и его
приходилось переделывать.

Карточка даёт: цветной бейдж формата, название, дату, размер, иконку
скачивания, у заголовка группы — счётчик файлов.

Исключение одно: компактная строка-ссылка внутри чужой плитки (например,
«Положение» в карточке подразделения в `StructureView`). Это не список
документов — полная карточка там неуместна.

## Как это устроено

- **Формат** — из расширения в адресе. Не хардкодить: файл может быть `.doc`,
  `.rtf`, `.xlsx`.
- **Дата и размер** — файлы лежат на orgma.ru, их метаданные знает только
  сервер источника. Собираются HEAD-запросами:
  - типовые страницы (`content/pages/**`) → `node scripts/pages/enrich-files.mjs`
    → `content/pages/file-meta.json`;
  - sveden → `node scripts/sveden/enrich-docs.mjs` → `content/sveden/doc-meta.json`.
  Результат коммитится: сборка на GitHub Pages в сеть не ходит.
  **Добавили файлы — прогоните скрипт**, иначе карточки будут без даты и размера.
- **Длинные названия.** Названия на orgma — это полные заголовки актов (до 900
  символов). `DocCard` режет их по первой «ёлочке»: голова (тип, дата, номер)
  видна всегда, название в кавычках свёрнуто до двух строк и раскрывается
  кнопкой. Кнопка раскрытия — **рядом со ссылкой, не внутри неё**: вложенные
  интерактивные элементы ломают клавиатуру и скринридеры.
- **Адреса файлов.** В yml держим читаемыми (пробелы и кириллица), кодирование —
  при рендере (`encodeFileHref`). Не кодировать в данных: легко получить
  полу-кодированный гибрид и двойное `%2520`.
- **Ссылки внутренние (`/files/...`)** — обычный `<a>`, basePath автоматически
  не применяется. Прогонять через `asset()`.
- **ЭЦП не показываем.** В макете плашка «Подписан ЭЦП» есть, но orgma не
  публикует подписи (`.sig`) — подтвердить нечем. Рисовать её было бы враньём.
- **Нет файла — не прячем строку.** `DocCard` показывает заготовку: серый бейдж,
  «—» вместо размера, без ссылки.

# Данные в YAML

Строка с **двоеточием и пробелом** в незакавыченном скаляре ломает YAML: в
списке она молча превращается в объект и роняет рендер. Такие значения
закавычивать. Загрузчики (`pages.ts`, `events.ts`) на это ругаются с указанием
файла — читайте сообщение, оно точное.

# Данные — только реальные

Не выдумывать данные и не переносить демо-значения из макетов (там вымышленные
ставки, зарплаты, номера закупок, ФИО). Нет данных — заготовка с прочерками и
комментарием в yml, кто заполняет. Ссылки на файлы проверять: на orgma
встречаются битые.
