# Handoff: сайт ОрГМУ → Next.js

Стек, выбранный для реализации: **TypeScript + App Router + Tailwind CSS** (эквивалент Next.js `create-next-app` с флагами `--typescript --tailwind --app --eslint`).

## Обзор
Полный флагманский сайт вуза: главная, разделы «Поступающим», «Сведения об образовательной организации», новости/мероприятия, каталог образовательных программ, структура и персоны, сервисные страницы (документы, доступная среда, стипендии, конкурсные списки и т.д.), плюс баннер cookie-согласия на всех страницах.

## О файлах в этом пакете
Всё в `pages/`, `components/`, `design-system/` — **HTML-прототипы (Design Components)**, референс, а не код для копирования 1:1. Задача разработчика — **пересобрать этот дизайн в Next.js**: маршруты `app/`, React-компоненты, Tailwind-классы (или CSS-переменные из `tokens.css`, если Tailwind не используется), реальные данные вместо моков из `staff-data.js` / `structure-data.js`.

Каждый `.dc.html` открывается напрямую в браузере (рядом лежит `support.js`, `tokens.css` не обязателен для рендера превью, но обязателен как источник значений).

## Fidelity
**Hi-fi.** Все цвета, отступы, шрифты, радиусы, тени — финальные, взяты из Figma-макета и брендбука. Значения зашиты в инлайн-стили самих `.dc.html` — сверяться нужно с исходным HTML, а не «на глаз» со скриншота.

## Структура пакета
- `pages/` — 29 экранов сайта (см. таблицу ниже).
- `components/` — переиспользуемые блоки: шапка, подвал, боковое меню, карточки, плитки, баннеры.
- `design-system/` — `Styleguide` (токены наглядно), `Brandbook` (бренд-гайд, логотип, цвета вуза), `Theming` (как темизируются подсайты: Клиника/НИЦ/Библиотека), `IA` (карта информационной архитектуры — полный список разделов и пунктов меню).
- `tokens.css` — единый источник цветов/шрифтов/отступов/радиусов/теней (CSS custom properties). Подключать как есть или переложить в `tailwind.config.ts`.
- `staff-data.js`, `structure-data.js` — моковые датасеты (сотрудники, оргструктура) — заменить на реальный источник данных (CMS/БД/API).
- `image-slot.js` — вспомогательный плейсхолдер для изображений (не переносить в прод, там используется реальный `<Image>`).
- `support.js` — рантайм для рендера `.dc.html` в браузере, разработчику не нужен, только для просмотра прототипов.
- `sitemap-index.html` — навигатор по всем страницам прототипа со ссылками (для быстрого просмотра).
- `assets/` — логотип, фото корпуса.

## Компоненты (`components/`)
- **SiteHeader** — шапка: лого, mega-меню (десктоп) / аккордеон (мобилка, брейкпоинт 1050px).
- **SideRail** — боковая колонка соцсетей/быстрых ссылок.
- **SiteFooter** — подвал: адрес, телефоны, 2 колонки ссылок, копирайт.
- **Banner**, **AnnouncementBar** — промо-баннер и полоса срочных объявлений.
- **CookieBanner** — ненавязчивый баннер cookie-согласия (см. ниже).
- **BannerSlider** — слайдер промо-баннеров (по 2 на экран), см. Changelog.
- **Tile**, **NewsCard**, **EventCard**, **ProgramTile**, **PersonTile**, **StatPair** — карточки/плитки для лент.
- **FilterSelect** — селект-фильтр для списков (события/программы/новости).

## Cookie-баннер — поведение
- Показывается один раз, внизу экрана, узкая плашка (не модалка, не блокирует контент), на **всех** страницах.
- Текст + ссылка на страницу `Privacy`, кнопка «Хорошо».
- После нажатия — скрывается и не показывается повторно: состояние пишется в `localStorage` (`orgma-cookie-consent`). В проде — тот же паттерн (localStorage или cookie на 6–12 мес.), плюс серверная проверка, если нужна юридическая фиксация согласия.
- Подключён на каждой странице после подвала: `<SiteFooter /><CookieBanner />`.

## Design Tokens
Источник — `tokens.css`. Ключевое:
- Брендовые цвета: `--c-brand` (основной синий #00659B), `--c-accent` (терракотовый, кнопки/ссылки), `--c-sky-soft`, нейтрали `--c-ink*`, `--c-bg*`, `--c-line*`.
- Подсайты переопределяют `--c-accent`/`--c-link` через `[data-subsite="clinic|science|library"]` — один визуальный язык, разный акцент.
- Шрифты: `Inter` (дисплей/заголовки), `Roboto Condensed` (UI/текст). Шкала размеров `--fs-display` (50px) … `--fs-2xs` (11px).
- Отступы: шкала на основе 4px (`--s-1` … `--s-20`).
- Радиусы `--r-sm…--r-pill`, тени `--sh-1…--sh-menu`, длительности переходов `--dur-fast/--dur/--dur-slow`.
- Брейкпоинты для медиа-запросов: `--bp-mobile` 430px, `--bp-tablet` 768px, `--bp-desktop` 1024px, `--bp-wide` 1440px.

При переносе в Tailwind — эти значения идут в `theme.extend` (`colors`, `fontSize`, `spacing`, `borderRadius`, `boxShadow`, `screens`), имена ключей можно сохранить как есть.

## Интерактивность / состояния
- **SiteHeader**: hover/click по пункту меню открывает mega-меню на десктопе; на мобилке — бургер + аккордеон с раскрытием секций.
- **Фильтры списков** (News/Events/ProgramCatalog/StaffDirectory/RatingLists) — клиентские селекты (`FilterSelect`), реальная фильтрация данных — на разработчика (сейчас на моках).
- **CookieBanner** — см. выше.
- Ховер-состояния ссылок/кнопок заданы инлайново (`style-hover` в DC-разметке = обычный `:hover` в CSS/Tailwind).
- Адаптив: у большинства страниц боковая колонка (`.rail-col`) скрывается на планшете/мобилке — смотри media-запросы в `<style>` каждого файла.

## Данные
- `staff-data.js` (`window.STAFF_DATA.people`) — единый датасет персон, один человек может встречаться и в «Руководстве», и в карточке кафедры.
- `structure-data.js` (`window.STRUCTURE_DATA`) — плоский список оргединиц с `parent`/`depth` для построения дерева, ссылается на `staff-data.js` через `headId`.
- В Next.js — эти два файла становятся моделью данных (типы + фикстуры на первое время, затем реальный API/CMS).

## Как смотреть
Открой `sitemap-index.html` — там ссылки на все страницы прототипа с кратким описанием каждой (нужен HTTP-хостинг локально, не `file://`, т.к. страницы подгружают общие блоки).

## Changelog
- **BannerSlider** (`components/BannerSlider.dc.html`) — новый компонент: слайдер промо-баннеров на главной, по 2 баннера на экран (1 на мобилке), стрелки + точки-пагинация, страницы листаются целиком. Данные — массив `banners` (`kicker`, `title`, `image`, `tone`), сейчас заглушка на 6 штук для теста; в проде — реальные баннеры из CMS/API. Встроен на `Home` вместо статичной пары баннеров.
- **Article** — добавлен слайдер фото в шапке новости/статьи (несколько фото на одну новость): стрелки, точки, счётчик, подпись под каждым фото. Данные — массив `photos` (`url`, `caption`).
- **Apps** — карточка ЭИОС теперь одна, с несколькими точками входа (основная кнопка + раскрывающийся список доп. ссылок, например «СДО · sdo.orgma.ru»), вместо дублирования на несколько карточек одного и того же приложения.
- **CookieBanner** — добавлен ненавязчивый баннер cookie-согласия на все страницы (см. описание компонента выше).
